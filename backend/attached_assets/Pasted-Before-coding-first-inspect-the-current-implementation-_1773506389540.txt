B) Basket gain/loss since response time
For each saved response, calculate the performance of the tickers/assets mentioned/recommended in that response as if they were bought at the time the response was generated.

Business logic:
- treat each saved response as an equal-weight basket of the extracted symbols
- entry time = response created_at
- current value = latest available price now
- basket return = mean of valid per-symbol returns
- if no valid symbols found, basket performance is unavailable
- if only some symbols are priceable, compute using available ones and expose partial status
