Hyperliquid Backend Data Contract
Base URL your FastAPI must expose:

GET  /api/hyperliquid/screener/snapshot?market_type=all&limit=200
POST /api/hyperliquid/screener/agent-rank
GET  /api/hyperliquid/screener/asset/{coin}
(These are the paths your frontend's Node proxy forwards to)

Critical rules
Never return 0 for a metric that is genuinely unavailable — always return null. The frontend displays — for null and treats 0 as a real value.

All percentage/rate fields are raw decimals — 0.053 means 5.3%, 0.0001 means 0.01%/hr. The frontend multiplies by 100 for display.

All USD amounts are full dollar values — 4200000000.0 not 4200 or 4.2B.

Score fields (volatility, momentum, breakoutScore, etc.) must be 0–1 normalized — the frontend colors them: ≥0.6 green, ≤0.35 red, otherwise amber.

signalDirection must be exactly "bullish", "bearish", or "neutral" — no other strings.

updatedAt must be ISO 8601 with a Z suffix: "2025-01-01T12:00:00Z".

Hyperliquid main info call to get all market data in one shot:

import httpx
resp = httpx.post("https://api.hyperliquid.xyz/info",
    json={"type": "metaAndAssetCtxs"})
meta, asset_ctxs = resp.json()  # meta is universe info, asset_ctxs is per-asset context
Each asset_ctxs[i] maps to meta["universe"][i] by index. Fields you'll use: markPx, oraclePx, funding, openInterest, dayNtlVlm, prevDayPx, impactPxs, premium.
