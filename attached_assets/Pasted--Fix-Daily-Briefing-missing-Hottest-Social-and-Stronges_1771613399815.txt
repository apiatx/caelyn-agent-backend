## Fix: Daily Briefing missing "Hottest Social" and "Strongest Sector" data

### Root Cause 1: "Hottest Social" = N/A

The `get_morning_briefing()` in `market_data_service.py` does NOT call Grok/xAI for X sentiment. The only social data comes from StockTwits trending (line 2498), which often returns empty or limited data. The "Hottest Social" scanner card has nothing to display.

**Fix: Add Grok X sentiment scan to the morning briefing**

In `get_morning_briefing()`, add the Grok call to the parallel tasks. Find the `briefing_tasks` list (around line 2485) and add:
```python
        # Add X sentiment if available
        x_task = None
        if self.xai:
            x_task = asyncio.wait_for(
                self.xai.get_trending_tickers("stock"),
                timeout=15.0
            )
```

Then add it to the gather. Since the existing gather uses positional unpacking, the cleanest approach is to run the X scan separately in parallel:
```python
        # Run X sentiment in parallel with the main briefing tasks
        x_sentiment_data = {}
        if self.xai:
            try:
                x_result = await asyncio.wait_for(
                    self.xai.get_trending_tickers("stock"),
                    timeout=15.0
                )
                if isinstance(x_result, dict) and not x_result.get("error"):
                    x_sentiment_data = x_result
                    print(f"[Briefing] X sentiment: {len(x_result.get('trending_tickers', []))} trending tickers")
                else:
                    print(f"[Briefing] X sentiment: error or empty")
            except asyncio.TimeoutError:
                print(f"[Briefing] X sentiment: timed out")
            except Exception as e:
                print(f"[Briefing] X sentiment failed: {e}")
```

### Root Cause 2: "Strongest Sector" = blank

FMP `get_sector_performance()` returns sector performance data. It's stored in `fmp_data["sector_performance"]` and returned as `fmp_market_data`. But Claude may not know to use this for the "Strongest Sector" card.

**Fix: Extract the top sector into pre_computed_highlights**

Find `_compute_signal_highlights()` method and add a strongest sector extraction. Or simpler — add it directly in the return:
```python
        # Extract strongest sector from FMP data
        strongest_sector = None
        sector_perf_data = fmp_data.get("sector_performance", [])
        if isinstance(sector_perf_data, list) and sector_perf_data:
            # Sort by changesPercentage descending
            sorted_sectors = sorted(
                [s for s in sector_perf_data if isinstance(s, dict) and s.get("changesPercentage")],
                key=lambda x: float(str(x.get("changesPercentage", "0")).replace("%", "")),
                reverse=True
            )
            if sorted_sectors:
                top = sorted_sectors[0]
                strongest_sector = {
                    "sector": top.get("sector", ""),
                    "change": top.get("changesPercentage", ""),
                }
                print(f"[Briefing] Strongest sector: {strongest_sector['sector']} ({strongest_sector['change']})")
```

Then add it to the return:
```python
        return {
            ...existing...
            "strongest_sector": strongest_sector,
        }
```
