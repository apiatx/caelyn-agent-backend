Exact instructions for your backend agent:

The GET /api/insider-activity endpoint needs two filter parameters to work correctly:

1. type parameter — Filter transactions by type. The frontend sends single-letter SEC Form 4 codes:

type=P → Purchase/Buy transactions
type=S → Sale transactions
type=M → Option exercises
type=G → Gift transactions
The endpoint must filter transaction_type (or whatever your field is named) against these values. If your database stores the full word (e.g. "Purchase" or "Buy"), you need to map P→Purchase, S→Sale etc. before querying, OR store and match on the single-letter codes.

2. min_score parameter — Filter transactions where score >= min_score. The frontend sends a number as a string (e.g. min_score=70). The endpoint must parse it as an integer and apply WHERE score >= 70 (or equivalent) to the results.

These two parameters are used together. Example call that must return results:

GET /api/insider-activity?type=P&min_score=70&sort=score&order=desc&limit=50
This should return buy transactions with a conviction score of 70 or higher, sorted best-first.
